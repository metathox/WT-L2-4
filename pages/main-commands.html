Загальні команди

Головна Основні поняття Наглядні приклади

Команди Git для взаємодії через термінал
(Потребує встановлення Git на ПК)

Команда <Аргумент> Опис Приклади
git init <directory> Створює та ініціалізує новий локальний репозиторій
(опціонально вказуємо директорію — себто теку)		 git init — створює новий репозиторій в поточній теці.
git clone <url> <directory> Копіює віддалений репозиторій
(атрибути: посилання на репозиторій, директорія в яку копіюємо)		 git clone https://github.com/user/repo.git — клонування у поточну теку
git clone https://github.com/user/repo.git myrepository — клонування в задану теку
git status Показує стан репозиторію Зміни у файлах, внесені зміни, що готове до коміту
git add <file> | <directory> | . Додає зміни у staging area / index (зона підготовки перед комітом) git add index.html — додає один файл
git add src/ — додає зміни у цілій теці
git add . — додає зміни у поточній теці
git commit -m <message> Вносить зміни до репозиторію (зазвичай окремої гілки) git commit -m "Додано нову секцію на головній сторінці"
git log [--graph] [--oneline] [--all] Показує історію комітів git log — детальна історія комітів
git log --oneline — коротка історія комітів
git log --graph --oneline --all — візуальна графічна історія всіх гілок
git branch <branch-name> Показує існуючі гілки або створює нову
(створює за допомогою атрибуту з назвою нової гілки)		 git branch — показує всі локальні гілки
git branch feature-login — створює нову гілку feature-login
git checkout <branch-name> | <commit-id> Перемикається між гілками або комітами git checkout main — перемикається на гілку main
git checkout a1b2c3d — перемикається на конкретний коміт
git merge <branch-name> Об'єднує гілки git checkout main, git merge feature-login — об’єднує зміни
з feature-login у main
git push <remote> <branch> Відправляє зміни на віддалений репозиторій
(атрибути: посилання, гілка репозиторію)		 git push origin main — відправляє гілку main на віддалений репозиторій origin
